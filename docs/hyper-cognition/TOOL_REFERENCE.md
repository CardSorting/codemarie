# Tool Reference: Hyper-Cognition & Swarm Memory

The integration adds a suite of `MEM_*` tools designed for deep repository analysis and coordinated work.

## Memory Management
- `MEM_QUERY`: Search the Knowledge Graph with recursive neighborhood augmentation.
- `MEM_SNAPSHOT`: Capture the current cognitive and structural state into a permanent landmark.
- `MEM_LINK`: Manually create semantic edges between knowledge nodes or files.
- `MEM_BUNDLE`: Prepare a high-density handoff package for a sub-agent.

## Swarm Coordination
- `MEM_CLAIM`: Acquire a persistent lock on a file or concept.
- `MEM_RELEASE`: Release a previously held lock.
- `MEM_APPEND_SHARED`: Post a global guideline or constraint to the swarm shared layer.

## Predictive Analytics
- `MEM_BLAST`: Calculate the recursive impact radius of a proposed change.
- `MEM_FORECAST`: Speculatively merge two task streams and predict semantic conflicts.
- `MEM_HUBS`: Identify high-centrality "chokepoint" nodes in the codebase.
- `MEM_CHOKE`: Identify files with high churn and contention.

## Self-Correction
- `MEM_HEAL`: Attempt to repair structural inconsistencies in the repository or graph.
- `MEM_REFRESH`: Update the confidence scores of decaying knowledge nodes.
